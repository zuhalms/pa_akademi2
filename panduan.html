1. Tentang Sistem SMART-BA

SMART-BA (Smart Bimbingan Akademik) adalah sistem informasi pembimbingan akademik berbasis web yang dirancang untuk memfasilitasi proses bimbingan akademik antara mahasiswa dan dosen pembimbing akademik (PA) di Fakultas Syariah UIN Palopo.

üéØ Tujuan Sistem

  • ‚úì Mendigitalkan proses bimbingan akademik (paperless)
  • ‚úì Memudahkan monitoring perkembangan akademik mahasiswa
  • ‚úì Menyediakan dokumentasi lengkap riwayat bimbingan
  • ‚úì Meningkatkan efisiensi komunikasi dosen-mahasiswa
  • ‚úì Mendukung Smart & Green Campus Initiative

Fitur Utama Sistem

üìö Logbook Digital

Pencatatan setiap sesi bimbingan secara online. Baik dosen maupun mahasiswa dapat menambahkan catatan kapan saja.

‚úÖ Pelacak Pencapaian

Visualisasi kemajuan studi dari Seminar Proposal hingga Yudisium dengan checklist interaktif.

üìÅ Upload Dokumen

Bagikan file penting seperti draft KRS atau revisi skripsi dengan mudah langsung dalam sistem.

‚≠ê Evaluasi Dua Arah

Dosen dapat mengevaluasi soft skill mahasiswa, dan mahasiswa dapat memberi umpan balik kinerja dosen PA.

6. Input Riwayat Akademik

Tujuan Input Riwayat Akademik

  • ‚úÖ Dokumentasi IP & SKS per semester secara sistematis
  • ‚úÖ Auto-calculate IPK berdasarkan weighted average
  • ‚úÖ Monitoring perkembangan akademik real-time
  • ‚úÖ Basis data untuk evaluasi dosen PA
  • ‚úÖ Generate chart progress di dashboard

Cara Input Riwayat Akademik

  1. Akses menu "Riwayat Akademik"
    Klik di sidebar atau menu "Input Riwayat"
  2. Lihat form 14 semester
    Sistem menyediakan form untuk semester 1 sampai 14
  3. Input data per semester
    Untuk setiap semester, masukkan:
    - IP Semester: contoh 3.45 atau 3,45
    - SKS Semester: contoh 21
  4. Sistem auto-calculate
    IPK dan Total SKS akan otomatis terhitung
  5. Klik "Simpan Data"
    Data tersimpan ke database dan profil terupdate

üìê Formula Perhitungan

IPK (Indeks Prestasi Kumulatif):

IPK = Œ£(IP √ó SKS) / Œ£(SKS)

Total SKS:

Total SKS = Œ£(SKS Semester 1-14)

Validasi Input

Field Validasi Contoh Valid
IP Semester 0.00 - 4.00 3.45 atau 3,45
SKS Semester 0 - 24 21, 20, 18
Format Desimal Koma atau titik 3.45 = 3,45

‚úÖ Update Data

Data riwayat akademik dapat diupdate kapan saja. Sistem menggunakan UPSERT operation (insert or update), jadi jika data sudah ada, akan di-update; jika belum ada, akan di-insert.

IPK dan Total SKS akan otomatis recalculate setiap kali Anda save.

7. Konsultasi Judul Penelitian

Fitur ini memungkinkan mahasiswa mengajukan judul penelitian/skripsi kepada dosen PA untuk mendapat persetujuan atau feedback.

Cara Mengajukan Judul

  1. Akses menu "Konsultasi Judul"
    Klik di sidebar mahasiswa
  2. Klik "Ajukan Judul Baru"
    Tombol atau form pengajuan akan muncul
  3. Isi form pengajuan
    - Judul Usulan: Tuliskan judul penelitian lengkap
    - Deskripsi/Latar Belakang: Jelaskan alasan pemilihan judul, rumusan masalah, dan tujuan penelitian
  4. Klik "Submit Pengajuan"
    Judul akan masuk ke dashboard dosen PA
  5. Tunggu tanggapan
    Dosen akan memberi feedback: Disetujui, Ditolak, atau Revisi

Status Konsultasi

Status Arti Action
Menunggu Respon Dosen belum menanggapi Tunggu atau follow up ke dosen
Disetujui Judul disetujui dosen Lanjut ke tahap berikutnya
Ditolak Judul tidak disetujui Baca alasan dan ajukan judul baru

Melihat Tanggapan Dosen

  1. Di halaman "Konsultasi Judul", scroll ke bawah
  2. Anda akan melihat History Konsultasi dengan detail:
    • Tanggal pengajuan
    • Judul yang diajukan
    • Deskripsi lengkap
    • Status (badge berwarna)
    • Tanggapan Dosen (jika sudah ditanggapi)
    • Tanggal respon

üí° Tips Mengajukan Judul

  • Ajukan judul dengan deskripsi yang jelas dan detail
  • Jelaskan latar belakang masalah dengan baik
  • Jika ditolak, baca alasan dosen dengan seksama
  • Revisi berdasarkan feedback dan ajukan kembali
  • Konsultasi tatap muka jika perlu diskusi lebih mendalam

9. Logbook Bimbingan

Apa itu Logbook?

Logbook adalah catatan digital dari setiap sesi bimbingan antara Anda dan dosen PA. Baik mahasiswa maupun dosen dapat menambahkan catatan, sehingga riwayat bimbingan terdokumentasi dengan lengkap.

  1. Scroll ke section "Logbook Bimbingan"
    Di Dashboard mahasiswa atau menu Logbook
  2. Klik "Tambah Catatan Bimbingan"
    Form input akan muncul
  3. Isi form

    - Isi Bimbingan: Detail pembahasan, pertanyaan, catatan penting

    Catatan tersimpan dan dosen akan menerima notifikasi

  • Buka Dashboard untuk melihat isi catatan dosen

21. FAQ (Frequently Asked Questions)

Q1: Lupa password, bagaimana?

A: Hubungi admin sistem atau bagian IT kampus untuk reset password. Bawa kartu identitas untuk verifikasi.

Q2: Upload file gagal terus, kenapa?

A: Pastikan:

  • Ukuran file tidak melebihi 5 MB
  • Format file sesuai (PDF, DOC, DOCX, JPG, PNG)
  • Koneksi internet stabil
  • Folder dokumen/ dan assets/uploads/ memiliki write permission

Q3: IPK tidak terupdate setelah input riwayat akademik?

A:

  • Pastikan Anda klik tombol "Simpan Data"
  • Refresh halaman (F5 atau Ctrl+R)
  • Jika masih gagal, logout dan login kembali

Q4: Notifikasi tidak muncul?

A:

  • Refresh dashboard
  • Clear browser cache
  • Pastikan browser JavaScript enabled

Q5: Mahasiswa tidak menerima notifikasi persetujuan KRS?

A: (Untuk Dosen)

  • Pastikan klik tombol Approve/Reject sampai selesai
  • Mahasiswa harus login ulang untuk melihat notifikasi

Q6: Tidak bisa hapus logbook?

A:

  • Pastikan Anda pemilik logbook (yang input)
  • Dosen PA dapat hapus logbook mahasiswa bimbingannya
  • Admin dapat hapus semua logbook

Q7: Foto profil tidak tampil?

A:

  • Pastikan file berhasil diupload (ada pesan sukses)
  • Hard refresh browser (Ctrl+Shift+R)
  • Cek folder assets/uploads/ apakah file ada

Q8: Laporan PDF tidak bisa dibuka?

A:

  • Pastikan ada aplikasi PDF reader (Adobe, browser PDF viewer)
  • Download file dulu, jangan buka langsung
  • Jika corrupt, generate ulang laporan

22. Troubleshooting

Masalah Login

‚ö†Ô∏è Gejala: "Username atau password salah" terus-menerus

Solusi:

  1. Pastikan username benar (NIM untuk mahasiswa, ID/NIP untuk dosen)
  2. Password case-sensitive, periksa CAPS LOCK
  3. Coba reset password via admin
  4. Coba browser lain (Chrome, Firefox, Edge)

‚ö†Ô∏è Gejala: Setelah login, langsung logout otomatis

Solusi:

  1. Clear browser cookies & cache
  2. Pastikan browser allow cookies
  3. Disable browser extensions yang block cookies
  4. Coba incognito/private mode

Masalah Database

‚ö†Ô∏è Gejala: Error "Koneksi database gagal"

Solusi (Admin/Developer):

  1. Cek file config.php - pastikan kredensial benar
  2. Pastikan MySQL service running
  3. Cek hostname: localhost atau 127.0.0.1
  4. Periksa user database memiliki privilege

‚ö†Ô∏è Gejala: Error "Table doesn't exist"

Solusi (Admin):

  1. Import database db_pa_akademi.sql dengan benar
  2. Periksa nama database di config.php
  3. Pastikan semua tabel sudah ter-create

Masalah Upload

‚ö†Ô∏è Gejala: "Failed to move uploaded file"

Solusi:

  1. Cek permission folder (chmod 777 untuk testing)
  2. Pastikan folder dokumen/ dan assets/uploads/ ada
  3. Periksa PHP upload_max_filesize dan post_max_size

Masalah Tampilan

‚ö†Ô∏è Gejala: Layout berantakan, CSS tidak load

Solusi:

  1. Hard refresh (Ctrl+Shift+R)
  2. Clear browser cache
  3. Cek koneksi internet (CDN Bootstrap)
  4. Periksa console browser untuk error (F12)

‚ö†Ô∏è Gejala: Chart tidak muncul di dashboard mahasiswa

Solusi:

  1. Pastikan ada data riwayat akademik
  2. Periksa JavaScript console error (F12)
  3. Pastikan ChartJS library loaded (lihat Network tab)
